# Dataset Plan

## Source Layers

| Layer | Source | Refresh |
|---|---|---|
| Card oracle | Scryfall `oracle_cards` | Daily or on demand |
| Printings/images | Scryfall `default_cards`, MTGJSON printings | Daily/weekly |
| Sets/products/decks | MTGJSON files | Daily/weekly |
| Combos | Commander Spellbook API/backend | Daily/weekly |
| Limited gameplay | 17Lands public datasets | Per public dump |
| Cubes/decklists | CubeCobra exports, user imports, MTGJSON deck files | Manual/import |
| Engine logs | Argentum/mage-bench runs | Per benchmark run |

## Ingestion Principles

Do not scrape unstable 17Lands UI endpoints for training. Use public dumps.

Do not redistribute card images or raw bundled card text in a public artifact. Store local caches for research and only publish derived benchmark records where the license/IP posture is acceptable.

Keep every source load reproducible:

| Field | Meaning |
|---|---|
| `source_id` | Source ledger id |
| `fetched_at` | UTC timestamp |
| `source_url` | Original URL or repo commit |
| `content_hash` | SHA-256 of raw file |
| `loader_version` | Version of our loader script |
| `row_count` | Count after parsing |

## Benchmark Datasets

### Card Knowledge

Generated from Scryfall/MTGJSON:

| Task | Label |
|---|---|
| legality classification | format legality |
| color identity | color identity |
| card type parsing | type/supertype/subtype |
| oracle retrieval | exact oracle/ruling span |
| set/product lookup | set metadata |

### Draft And Limited

Generated from 17Lands public dumps:

| Task | Label |
|---|---|
| pack 1 pick 1 prediction | human pick |
| contextual pick prediction | human pick given pool/history |
| deck build from pool | registered deck |
| archetype inference | deck colors/archetype |
| game outcome value | win/loss, turn outcome |

The key evaluation should not be raw top-1 accuracy only. Use regret-style scoring: how much worse was the selected card compared with observed expert/card-performance alternatives?

### Combo Reasoning

Generated from Commander Spellbook:

| Task | Label |
|---|---|
| identify combo pieces | variant cards |
| explain output | produced features |
| line ordering | normalized steps, when available |
| deck combo discovery | variants found in input card list |

Engine-verifiable combo execution should be a later subset, because not every combo is supported by a given rules engine.

### Engine Microstates

Generated manually at first from Argentum-supported cards, then from mage-bench golden tests:

| Task | Label |
|---|---|
| legal-action mask | engine legal actions |
| state transition | post-step digest/state |
| lethal detection | win/loss after action |
| stack interaction | resolved object/state |
| combat math | damage/life/permanent changes |

### Full Games

Generated by Argentum self-play and mage-bench/XMage:

| Artifact | Use |
|---|---|
| raw game JSONL | replay/debug |
| per-decision observations | policy training/eval |
| legal actions | action mask |
| final outcome | reward |
| blunder annotations | diagnostic metric, not absolute truth |
